# Contributing to HG PPC

## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a new branch for your feature or bugfix
4. Install dependencies with `pnpm install`

## Development Workflow

- Use `pnpm dev` to start development servers
- Run `pnpm lint` before committing to ensure code quality
- Write tests for new features

## Commit Message Guidelines

- Use conventional commits format
- Prefix commits with:
  - `feat:` for new features
  - `fix:` for bug fixes
  - `chore:` for maintenance tasks
  - `docs:` for documentation changes

## Submitting a Pull Request

1. Ensure all tests pass
2. Describe the purpose of your changes in the PR description
3. Reference any related issues

## Code of Conduct

- Be respectful and inclusive
- Collaborate constructively
- Prioritize the project's goals

## Questions?

Open an issue or contact the maintainers.
