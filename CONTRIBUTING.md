# Contributing to HG PPC

## Getting Started

1. Fork the repository
2. Create a new branch for your feature or bugfix
3. Make your changes
4. Run tests and linters before submitting

## Development Setup

1. Install dependencies:
```bash
pnpm install
```

2. Run development server:
```bash
pnpm dev
```

## Code Guidelines

- Follow existing code style
- Write tests for new functionality
- Keep pull requests focused and concise
- Document any new features or API changes

## Commit Messages

- Use clear, descriptive commit messages
- Format: `type(scope): description`
- Types: `feat`, `fix`, `docs`, `chore`, `refactor`, `test`

## Submitting Pull Requests

1. Ensure all tests pass
2. Run linters
```bash
pnpm lint
```
3. Describe changes in the PR description
4. Link any related issues

## Reporting Issues

- Use GitHub Issues
- Provide detailed description
- Include reproduction steps
- Share relevant error messages or logs

## Code of Conduct

- Be respectful
- Collaborate constructively
- Welcome diverse perspectives

## Questions?

Contact the maintainers or open an issue for discussion.
